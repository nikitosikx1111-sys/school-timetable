# School Schedule Management System

Цей проєкт є базовою системою керування шкільним розкладом.  Він створений у рамках навчального завдання і демонструє, як можна зберігати зв'язану інформацію про предмети, вчителів, класи та учнів.

## Основні моделі

У репозиторії визначено чотири моделі зі зв'язками **`ForeignKey`** для представлення шкільних структур:

| Модель | Опис | Зв'язок |
|-------|------|---------|
| **Subject** | Зберігає дані про навчальний предмет (назва, короткий опис). | – |
| **Teacher** | Представляє вчителя. Вчитель має ім’я та відноситься до певного предмету. | `ForeignKey` на `Subject` |
| **SchoolClass** | Описує клас (наприклад, "5‑А" або "10‑B"). | – |
| **Student** | Зберігає дані про учня з посиланням на клас, у якому він навчається. | `ForeignKey` на `SchoolClass` |

## Структура проекту

- `models.py` — містить реалізацію моделей у вигляді класів Django. Якщо ви не користуєтесь Django, ви можете адаптувати ці класи під власний ORM або зберігати дані у базі даних за допомогою SQL.
- `README.md` — поточний файл, що пояснює призначення проєкту і описує його складові.

## Використання

1. **Створення середовища.** Встановіть залежності проєкту. У цьому прикладі використано Django, тому у файл `requirements.txt` додано цю бібліотеку:

   ```bash
   pip install -r requirements.txt
   ```

2. **Створення міграцій.** Якщо ви використовуєте Django, ініціалізуйте проєкт і застосуйте міграції:

   ```bash
   django-admin startproject core .
   python manage.py makemigrations
   python manage.py migrate
   ```

3. **Використання моделей.** Після налаштування середовища ви зможете створювати записи для предметів, вчителів, класів та учнів через Django shell або адміністративну панель.

## Розширення

Це лише початок роботи над системою розкладу. Ви можете додати додаткові таблиці та функціонал, наприклад розклад занять, кабінети, плани уроків, систему авторизації та різні типи користувачів (адміністратор, учитель, учень).
